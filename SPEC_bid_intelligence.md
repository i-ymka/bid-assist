# SPEC: Bid Intelligence — Расширенная аналитика + AI-анализ недели

## Суть

После просмотра `/bidstats` за неделю пользователь получает расширенные данные о победителях и своём профиле, видит время реакции на проект, а кнопка "Analyse" запускает AI-анализ всех данных с готовыми предложениями по улучшению — результат отображается в Telegram и публикуется как GitHub Issue.

---

## Что входит в скоуп

### A. Time to bid — время реакции

На каждой карточке лосса (и WIN) показываются два времени:
- **Моё время реакции**: минуты/часы от публикации проекта до моего бида
- **Время реакции победителя**: то же для выигравшего бида (только LOSS-карточки)

Формат: `⏱ You: 4 min | Winner: 12 min` (или `2h 15m` если > 1 часа).

Вычисляется из `bid.submitdate − project.time_submitted` (оба поля уже доступны в API). Кэшируется в `bid_outcomes` как два новых поля: `my_time_to_bid_sec` и `winner_time_to_bid_sec`.

### B. Расширенный профиль победителя в карточках LOSS

Дополнительно к уже отображаемым данным (username, country, rating, reviews, completion_rate) в каждой LOSS-карточке показываются:

| Поле | Примечание |
|------|-----------|
| Часовая ставка ($/hr) | `user.hourly_rate` из `/users/0.1/users/{id}/` |
| Лет на платформе | Вычисляется из `user.registration_date` |
| Earnings score (0–10) | `reputation.entire_history.earnings_score` |
| Job success rate (%) | `reputation.entire_history.completion_rate` (уже есть, отображать явнее) |
| Количество портфолио-работ | Отдельный запрос `get_portfolios(user_id)` → `total_count` |

**Исключены из скоупа v1 (нет официального API):** экзамены Freelancer Skills Lab, certifications count. Записаны как будущее улучшение в конце этой спеки.

Все новые поля кэшируются в расширенных колонках `bid_outcomes` — повторный `/bidstats` не делает новых API-запросов.

### C. Мой профиль в /bidstats (один раз в начале)

В шапке вывода `/bidstats weekly` показывается блок **"My profile"** с теми же полями что у победителей, плюс текущий bid adjustment и min daily rate. Запрашивается один раз через `/users/0.1/self/` + `get_portfolios(my_id)`. Не кэшируется в DB — обновляется при каждом запросе `/bidstats`.

### D. Кнопка "📊 Analyse week" и AI-анализ

После вывода карточек лоссов (или дашборда если лоссов нет) появляется кнопка **"📊 Analyse week"**.

При нажатии:
1. Бот отвечает: `⏳ Analysing your bids...`
2. Gemini CLI получает промпт с полным пакетом данных за неделю:
   - Все WIN: сумма, текст бида, timing
   - Все LOSS: сумма, текст бида, timing, профиль победителя (все поля из B)
   - Мой профиль (все поля из C)
   - Текущие настройки: bid_adjustment, min_daily_rate, prompts_dir
3. Gemini возвращает структурированный анализ:
   - Паттерны побед vs проигрышей (цена, скорость, длина предложения, профиль победителя)
   - Конкретные предложения по улучшению (промпт, ценообразование, скорость реакции, категории)
   - Приоритизированный список изменений (что важнее)
4. Анализ отправляется в Telegram как HTML-сообщение
5. Тот же анализ публикуется как GitHub Issue:
   - Заголовок: `[AI Analysis] Week of YYYY-MM-DD — @account_username`
   - Метка: `ai-analysis`
   - Тело: полный текст анализа + ссылка на неделю
   - Репозиторий: `GITHUB_REPO` из `.env` (например `username/bid-assist`)
   - Токен: `GITHUB_TOKEN` из `.env`

### E. Файл `docs/PROMPT_LOG.md`

Один общий файл для всех аккаунтов. Структура записи:

```markdown
## YYYY-MM-DD | Аккаунт: ymka | Win rate до: 11% → после: ?%

**Изменение:** что именно изменили в промпте или коде
**Мотивация:** почему (ссылка на GitHub Issue если есть)
**Результат:** заполняется через ~2 недели после изменения
**AI-предложения применены:** [Issue #N] краткое описание
```

Файл создаётся вручную (шаблон ниже), но бот при публикации GitHub Issue добавляет в него ссылку на новый Issue автоматически.

## Что НЕ входит (явно)

- Автоматическое применение AI-предложений к промптам или коду — только публикация и отображение
- Экзамены Freelancer Skills Lab и certifications count — нет официального API (задокументировано как future improvement)
- Анализ оба аккаунта одновременно — кнопка появляется в каждом боте отдельно
- Автоматический запуск анализа по расписанию — только по кнопке
- Хранение истории AI-анализов в SQLite DB
- Исторические тренды за несколько недель

---

## Критерии готовности

1. В каждой LOSS-карточке `/bidstats weekly` есть строка `⏱ You: Xmin | Winner: Ymin` с реальными данными (не нулями).

2. В каждой LOSS-карточке видны: hourly rate победителя, лет на платформе, earnings score (0-10), portfolio count.

3. В шапке `/bidstats weekly` (до карточек) — блок "My profile" с теми же полями о себе.

4. После последней карточки появляется кнопка **"📊 Analyse week"**.

5. Нажатие кнопки → в течение 120 секунд в Telegram появляется сообщение с анализом, содержащим минимум 3 пронумерованных предложения по улучшению.

6. В GitHub репозитории появляется новый Issue с заголовком `[AI Analysis] Week of YYYY-MM-DD`, меткой `ai-analysis`, и теми же предложениями в теле.

7. Файл `docs/PROMPT_LOG.md` существует с шаблоном и инструкцией по заполнению.

8. Повторный запрос `/bidstats` для уже классифицированных проектов не делает новых запросов к Freelancer API (данные из кэша `bid_outcomes`).

## Решённые вопросы

1. **AI для анализа** → Gemini CLI (уже настроен, ничего нового)
2. **GitHub** → `GITHUB_TOKEN` и `GITHUB_REPO` в `.env.yehia` / `.env.ymka` (gitignored). Токен — Personal Access Token с правом `repo`. Публикует Issues в репозиторий из `GITHUB_REPO`.
3. **Экзамены Freelancer** → нет официального API endpoint. Исключены из v1. Future improvement: попробовать `qualification_details=true` на users endpoint и проверить что возвращает.
4. **PROMPT_LOG** → один общий `docs/PROMPT_LOG.md` для всех аккаунтов.

## Future improvements (вне скоупа v1)

- Автоматическое добавление записи в PROMPT_LOG при публикации GitHub Issue
